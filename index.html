KI-Experiment: Code zu Excel für die Erstellung eines digitalen Zwillings in AnyLogic (unter Verwendung von GitHub Copilot)

Experiment: Der Filialleiter einer großen Supermarktkette hat uns damit beauftragt, einen digitalen Zwilling seines Marktes zu erstellen. Ziel ist es, den Eingangsbereich des Marktes an einem Montag zu simulieren. An einem typischen Montag betreten insgesamt 5434 Kunden den Markt, wobei die Verteilung dieser Kunden einer Glockenkurve ähnelt. Die Öffnungszeiten des Marktes sind von 07:00 bis 21:00 Uhr.

  • Ansatz Arrival Rate

  • Ansatz: Arrival Table in Database

Problem: AnyLogic bietet die Möglichkeit, Tabellen zu erstellen, jedoch sind diese ohne Werte. Es gibt keine einfache Funktion, die es ermöglicht, Tabellen automatisch mit Werten nach einem bestimmten Algorithmus zu füllen. Theoretisch könnten wir einen Algorithmus in Java programmieren, der bis zur Zahl XYZ zählt und die Werte nach einem bestimmten Muster verteilt, um die Agenten entsprechend zu erzeugen. Allerdings ist der Zeitaufwand hierfür sehr hoch, besonders für Laien oder Programmieranfänger.

Lösung: Die Nutzung von KI-Tools wie GitHub Copilot bietet eine effiziente Alternative. GitHub Copilot stellt Code zur Verfügung, der die Erstellung von Excel-Tabellen ermöglicht. Dieses Tool verfügt über eine Chatfunktion und erfordert keine tiefgehenden Programmierkenntnisse.

  • GitHub Copilot GUI

In dem Experiment wurde die Entwicklungsumgebung „Visual Studio Code“ genutzt, damit ist die Integration von GitHub Copilot möglich. In Visual Studio muss das Plugin „GitHub Copilot“ installiert sein und um es zu nutzen, ist eine Registrierung notwendig (Testversion von 30 Tagen verfügbar). Zusätzlich muss Python auf dem Arbeitsrechner installiert sein.

      Um diese Aufgabe zu erfüllen, können wir die folgenden Schritte ausführen:

      1. Erstellen Sie ein DataFrame, das alle Minuten des Tages von 06:00 bis 21:59 enthält.
      1. Generieren Sie eine Glockenkurve (Normalverteilung) mit der gleichen Anzahl von Punkten wie die Anzahl der Minuten.
      1. Skalieren Sie die Glockenkurve so, dass die Summe der Werte 5434 entspricht.
      1. Verteilen Sie die Kunden auf die Zeitintervalle basierend auf der Glockenkurve.
      1. Speichern Sie das DataFrame als Excel-Datei.

      Dieser Code erstellt eine Excel-Datei namens "Supermarkt_Kundenverteilung.xlsx", die die Verteilung der Kunden über den Tag darstellt. Die Kundenanzahl für jedes Zeitintervall ist auf ganze Zahlen gerundet und die Summe der Kundenanzahl entspricht 5434.

      Hier ist der Python-Code, der diese Schritte ausführt:

      import pandas as pd
import numpy as np
from scipy.stats import norm

# Gesamtzahl der Kunden
total_customers = 5434

# Erstellen Sie ein DataFrame mit allen Minuten des Tages von 06:00 bis 21:59
time_range = pd.date_range(start="06:00", end="21:59", freq='T')
df = pd.DataFrame(time_range, columns=['Datum und Uhrzeit'])

# Generieren Sie eine Glockenkurve (Normalverteilung)
x = np.linspace(0, len(df)-1, len(df))
distribution = norm.pdf(x, np.mean(x), np.std(x))

# Skalieren Sie die Glockenkurve so, dass die Summe der Werte 5434 entspricht
scaled_distribution = distribution / distribution.sum() * total_customers

# Verteilen Sie die Kunden auf die Zeitintervalle basierend auf der Glockenkurve
# Runden Sie die Werte auf ganze Zahlen und stellen Sie sicher, dass die Summe 5434 entspricht
df['Kundenanzahl'] = np.round(scaled_distribution).astype(int)
df.loc[df['Kundenanzahl'] > 10, 'Kundenanzahl'] = 10
remaining = total_customers - df['Kundenanzahl'].sum()
df.loc[df['Kundenanzahl'] < 10, 'Kundenanzahl'] += 1
df['Kundenanzahl'] = df['Kundenanzahl'] - (df['Kundenanzahl'].sum() - total_customers)

# Speichern Sie das DataFrame als Excel-Datei
df.to_excel('Supermarkt_Kundenverteilung.xlsx', index=False)

      import pandas as pd
import numpy as np
from scipy.stats import norm

# Gesamtzahl der Kunden
total_customers = 5434

# Erstellen Sie ein DataFrame mit allen Minuten des Tages von 06:00 bis 21:59
time_range = pd.date_range(start="06:00", end="21:59", freq='T')
df = pd.DataFrame(time_range, columns=['Datum und Uhrzeit'])

# Generieren Sie eine Glockenkurve (Normalverteilung)
x = np.linspace(-3, 3, len(df))
distribution = norm.pdf(x, 0, 1)

# Skalieren Sie die Glockenkurve so, dass die Summe der Werte 5434 entspricht und die Werte zwischen 0 und 10 liegen
scaled_distribution = distribution / distribution.sum() * total_customers
scaled_distribution = np.clip(scaled_distribution, 0, 10)

# Runden Sie die Werte auf ganze Zahlen und stellen Sie sicher, dass die Summe 5434 entspricht
df['Kundenanzahl'] = np.round(scaled_distribution).astype(int)
difference = total_customers - df['Kundenanzahl'].sum()
df.iloc[:difference, df.columns.get_loc('Kundenanzahl')] += 1

# Speichern Sie das DataFrame als Excel-Datei
df.to_excel('Supermarkt_Kundenverteilung.xlsx', index=False)

Fazit: GitHub Copilot eignet sich hervorragend, um Code zu generieren. Dennoch sollte sich der Anwender nicht vollständig auf GitHub Copilot verlassen. Die generierten Daten müssen getestet werden. Die Erstellung der Glockenkurvenverteilung stellte für die KI keine Herausforderung dar, jedoch war die Verteilung der Kundenanzahl von 5434 eine Hürde.
